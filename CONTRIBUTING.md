
# Contributing to gfgLock

Thank you for considering contributing to **gfgLock**! We welcome all improvements, bug fixes, new features, and documentation updates.

---

## üöÄ How to Contribute

### 1. Fork the Repository

Go to the main repository and click **Fork** to create your own copy.

### 2. Clone Your Fork

```bash
git clone https://github.com/<your-username>/gfgLock.git
cd gfgLock
```

### 3. Add Upstream Remote

```bash
git remote add upstream https://github.com/ShahFaisalGfG/gfgLock.git
git remote -v
```

### 4. Sync Your Local development Branch

```bash
git fetch upstream
git checkout development
git pull upstream development
```

### 5. Create a Feature Branch

```bash
git checkout -b feature-xyz
```

Use a clear branch name such as `feature-encryption-progress`, `bugfix-null-error`, etc.

### 6. Make Your Changes

Write clean, documented code and follow existing patterns.

### 7. Commit Your Work

```bash
git add .
git commit -m "Add: detailed explanation of the change"
```

### 8. Push to Your Fork

```bash
git push origin feature-xyz
```

### 9. Create a Pull Request (PR)

On GitHub, open a PR:

- **base branch:** development
- **compare:** your feature branch

Provide a clear description of the change.
